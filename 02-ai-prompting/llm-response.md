# Сырой ответ ИИ (LLM Response)

*Ответ сгенерирован LLM на промпт из [prompt.md](prompt.md). Скопирован без изменений.*

---

## Чек-лист тестирования формы «Заявка на перевозку»

### 1. Поля ввода

#### Город отправления / Город назначения (автокомплит)

| № | Проверка | Ожидаемый результат | Тип |
|---|----------|---------------------|-----|
| 1.1 | Выбрать существующий город из выпадающего списка | Город отображается в поле | Позитивный |
| 1.2 | Начать ввод названия города — проверить, что появляются подсказки | Автокомплит показывает совпадения | Позитивный |
| 1.3 | Ввести несуществующий город | Автокомплит не показывает результатов или показывает сообщение «Ничего не найдено» | Негативный |
| 1.4 | Оставить поле пустым и попытаться отправить форму | Ошибка валидации: «Поле обязательно для заполнения» | Негативный |
| 1.5 | Ввести специальные символы (XSS-инъекция: `<script>alert(1)</script>`) | Поле не принимает или экранирует ввод. Нет выполнения скрипта | Негативный |
| 1.6 | Выбрать город, очистить поле, попробовать отправить | Ошибка валидации | Негативный |
| 1.7 | Ввести 1 символ | Автокомплит должен показывать совпадения (если min length = 1) | Граничный |

#### Вес груза

| № | Проверка | Ожидаемый результат | Тип |
|---|----------|---------------------|-----|
| 2.1 | Ввести корректный вес (например, 5000 кг) | Поле принимает значение | Позитивный |
| 2.2 | Ввести вес = 1 кг | Поле принимает значение | Граничный |
| 2.3 | Ввести вес = 20 000 кг | Поле принимает значение (максимум) | Граничный |
| 2.4 | Ввести вес = 20 001 кг | Ошибка: превышение максимального значения | Негативный |
| 2.5 | Ввести вес = 0 кг | Ошибка: значение должно быть больше 0 | Негативный |
| 2.6 | Ввести отрицательное значение (-100) | Ошибка: отрицательное значение не допускается | Негативный |
| 2.7 | Ввести дробное значение (1500.5 кг) | Поле принимает значение (если дробные разрешены) | Позитивный |
| 2.8 | Оставить поле пустым | Ошибка: обязательное поле | Негативный |
| 2.9 | Ввести буквы/текст | Ошибка: поле принимает только числа | Негативный |

#### Объём груза

| № | Проверка | Ожидаемый результат | Тип |
|---|----------|---------------------|-----|
| 3.1 | Ввести корректный объём (например, 20 м³) | Поле принимает значение | Позитивный |
| 3.2 | Ввести объём = 1 м³ | Поле принимает значение | Граничный |
| 3.3 | Ввести объём = 82 м³ | Поле принимает значение (максимум) | Граничный |
| 3.4 | Ввести объём = 83 м³ | Ошибка: превышение максимального значения | Негативный |
| 3.5 | Ввести объём = 0 м³ | Ошибка: значение должно быть больше 0 | Негативный |
| 3.6 | Ввести отрицательное значение (-5) | Ошибка: отрицательное значение не допускается | Негативный |
| 3.7 | Ввести дробное значение (10.5 м³) | Поле принимает значение | Позитивный |
| 3.8 | Оставить поле пустым | Ошибка: обязательное поле | Негативный |

#### Дата погрузки

| № | Проверка | Ожидаемый результат | Тип |
|---|----------|---------------------|-----|
| 4.1 | Выбрать завтрашнюю дату | Дата принимается | Позитивный |
| 4.2 | Выбрать сегодняшнюю дату | Дата принимается | Позитивный |
| 4.3 | Попробовать выбрать вчерашнюю дату | Датапикер не позволяет выбрать прошедшую дату | Негативный |
| 4.4 | Выбрать дату через месяц | Дата принимается | Позитивный |
| 4.5 | Попробовать ввести текстовые символы в поле даты | Поле не принимает текст (защита на уровне датапикера) | Негативный |
| 4.6 | Не заполнять дату и нажать «Рассчитать» | Ошибка: обязательное поле | Негативный |

### 2. Кнопка «Рассчитать стоимость»

| № | Проверка | Ожидаемый результат | Тип |
|---|----------|---------------------|-----|
| 5.1 | Заполнить все поля валидными данными, нажать «Рассчитать» | POST-запрос отправлен, отображается цена | Позитивный |
| 5.2 | Нажать «Рассчитать» с незаполненными полями | Кнопка неактивна или показываются ошибки валидации | Негативный |
| 5.3 | Нажать «Рассчитать» несколько раз подряд | Обрабатывается только один запрос или показывается индикатор загрузки | Негативный |

### 3. API и обработка ошибок

| № | Проверка | Ожидаемый результат | Тип |
|---|----------|---------------------|-----|
| 6.1 | API возвращает 200 OK с ценой | Цена отображается в интерфейсе | Позитивный |
| 6.2 | API возвращает 400 Bad Request при некорректных данных | Отображается сообщение об ошибке | Негативный |
| 6.3 | API возвращает 404 Not Found (маршрут не найден) | Сообщение: «Маршрут не найден» | Негативный |
| 6.4 | API возвращает 500 ошибку | Отображается сообщение: «Ошибка сервера, попробуйте позже» | Негативный |
| 6.5 | API не отвечает (таймаут) | Отображается индикатор загрузки, затем сообщение об ошибке | Негативный |
| 6.6 | Проверить тело POST-запроса | Отправляются корректные параметры (города, вес, объём, дата) | Позитивный |
| 6.7 | Проверить, что API не сохраняет заявку (это только расчёт) | Только расчёт, без создания заказа | Позитивный |

### 4. UX и доступность

| № | Проверка | Ожидаемый результат | Тип |
|---|----------|---------------------|-----|
| 7.1 | Проверить состояние кнопки во время загрузки | Кнопка disabled, отображается спиннер | UX |
| 7.2 | Проверить отображение ошибки после неудачного расчёта | Сообщение об ошибке видно и понятно | UX |
| 7.3 | Проверить, что после успешного расчёта можно изменить параметры и пересчитать | Форма не блокируется, можно пересчитать | UX |
| 7.4 | Проверить клавиатурную навигацию (Tab, Enter) | Можно пройти по всем полям и нажать кнопку | UX |
| 7.5 | Проверить, что поле «Вес» принимает только цифры (numeric keyboard на мобильных) | Поле с type="number" или атрибутом inputmode="numeric" | UX |
| 7.6 | Проверить формат отображения цены | Цена в рублях, с разделителями разрядов | UX |
