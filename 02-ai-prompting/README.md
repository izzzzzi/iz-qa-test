## 1. Мой промпт

**Файл:** [prompt.md](prompt.md)

Промпт составлен по принципу **Role → Context → Task → Constraints → Output Format**:
- **Role:** QA-инженер, составляющий чек-лист
- **Context:** описание формы, бизнес-логика (автокомплит из БД, датапикер, POST-запрос)
- **Task:** позитивные, негативные сценарии, граничные значения
- **Constraints:** максимум 20000 кг, 82 м³, запрет прошлых дат
- **Output Format:** структурированный чек-лист с категориями

## 3. Ревью ответа ИИ

### 3.1. Что ИИ сделал хорошо

- ✅ **Структурировал чек-лист по категориям** (поля ввода, кнопка, негативные сценарии) — удобно читать
- ✅ **Покрыл happy path** — корректное заполнение всех полей, успешный расчёт
- ✅ **Указал граничные значения** — вес 0, 1, 19999, 20000, 20001; объём 0, 1, 81, 82, 83
- ✅ **Проверил датапикер** — прошлая дата, сегодня, завтра
- ✅ **Упомянул HTTP статус-коды** для кнопки расчёта
- ✅ **Добавил UX-проверки** — автокомплит, клавиатура (только цифры)

### 3.2. Где ИИ ошибся

#### ❌ Ошибка 1: Не учёл специфику автокомплита

ИИ написал «выбрать город из выпадающего списка» — но не проверил:
- **Ввод частичного названия** — должен работать поиск по подстроке
- **Выбор из выпадающего списка vs ввод кастомного текста** — можно ли ввести город, которого нет в БД? Если можно — что вернёт API?
- **Одинаковые названия городов** — есть ли дисклеймер с регионом (Воронеж обл. vs Воронеж Краснод. кр.)?
- **Пустой результат поиска** — ввёл "абв" и ничего не нашлось — что показывает UI?
- **Очистка поля после выбора** — можно ли сбросить выбор?

#### ❌ Ошибка 2: Не учёл связку город-отправления / город-назначения

ИИ проверяет города по отдельности, но не проверяет:
- **Отправление = Назначение** — перевозка из Москвы в Москву. Должна ли система возвращать ошибку?
- **Международные перевозки** — город в РФ → город за пределами РФ — работает ли логика?
- **Направления без тарифа** — из пункта А в пункт Б маршрут не найден (это упомянуто) — **что именно показывает API и UI?**

#### ❌ Ошибка 3: Нет проверок негативных сценаев API

ИИ написал «API возвращает ошибку, если маршрут не найден» — но не указал:
- **Таймаут API** — что если сервер расчёта завис на 30+ секунд?
- **500 ошибка API** — показываем тостер или форму сбрасываем?
- **429 Too Many Requests** — при частых нажатиях «Рассчитать»
- **Сетевая ошибка** — пользователь нажал «Рассчитать» и потерял интернет

#### ❌ Ошибка 4: Не учёл race condition

Пользователь:
1. Выбрал город отправления
2. Начал менять город назначения
3. НЕ ДОЖИДАЯСЬ загрузки автокомплита, ткнул «Рассчитать»

Или двойное нажатие на «Рассчитать» — отправляется 2 POST-запроса.

#### ❌ Ошибка 5: Объём груза — нет проверки на перевозку только по весу

В реальной логистике бывают перевозки:
- Только по весу (объём не указан)
- Только по объёму (вес не указан)
- По фактическому объёму/весу

ИИ не проверил **незаполненное поле объёма при заполненном весе** — должна ли форма разрешать это?

#### ❌ Ошибка 6: Дата погрузки — не учёл календарные границы

ИИ проверил «сегодня», «завтра», «вчера» — но не проверил:
- **Дата более чем через N месяцев/год** — есть ли ограничение на горизонт планирования?
- **Праздничные/нерабочие дни** — можно ли выбрать 1 января или 31 февраля?
- **Формат даты** — MM/DD/YYYY vs DD.MM.YYYY — зависит от региона

#### ❌ Ошибка 7: Избыточные проверки

ИИ добавил «проверить, что поле "Дата погрузки" не принимает текстовые символы» — для датапикера это избыточно. Датапикер по определению не даст ввести текст, если это нативный календарь, а не текстовое поле с маской.

Также избыточно: «проверить XSS-инъекции в поле города» — на этапе MVP с автокомплитом из БД это нецелесообразно (данные из БД, не от пользователя напрямую).

### 3.3. Сводка

| Категория | Что ИИ сделал хорошо | Что упустил | Что лишнее |
|-----------|---------------------|-------------|------------|
| Поля ввода | Типы полей, обязательность | Связка городов, частичный ввод в автокомплит | XSS-инъекции |
| Вес/объём | Граничные значения | Незаполненные поля, комбинации | — |
| Дата | Запрет прошлых дат | Горизонт планирования, формат | Текстовый ввод в датапикер |
| API | Статус-коды | Таймауты, race condition, 429, сетевые ошибки | — |
| UX | Клавиатура, формат | Очистка полей, сброс, двойное нажатие | — |

**Вывод:** ИИ даёт хорошую базовую структуру, но **глубинная бизнес-логика и edge cases** полностью ложатся на инженера. Без понимания предметной области (логистика) ИИ генерирует «общие» проверки, которые пропускают специфичные для домена сценарии.

## 4. Граничные условия для тестирования

**Файл:** [prompt.md](prompt.md) (внутри файла, раздел Boundary Analysis)

### 4.1. Город отправления / Город назначения

| Условие | Ожидаемый результат |
|---------|---------------------|
| Разные города из БД | Форма проходит валидацию |
| Отправление = Назначение | Ошибка: «Города отправления и назначения должны различаться» |
| Город не найден в БД | Ошибка: «Выберите город из списка» |
| Пустое поле | Ошибка: обязательное поле |
| Спецсимволы в поиске | Автокомплит показывает пустой список |
| Поиск по 1 символу | Автокомплит показывает совпадения (или ограничение min 2 символа) |

### 4.2. Вес груза

| Условие | Ожидаемый результат |
|---------|---------------------|
| 1 кг | Ок |
| 20 000 кг | Ок (граничное значение) |
| 20 001 кг | Ошибка: превышение макс. веса |
| 0 кг | Ошибка: вес должен быть > 0 |
| -1 кг | Ошибка: отрицательное значение |
| 10 000.5 кг (дробный) | Ок, если API поддерживает дробный вес |
| Пустое поле | Ошибка: обязательное поле |
| Буквы/символы | Ошибка: только цифры |

### 4.3. Объём груза

| Условие | Ожидаемый результат |
|---------|---------------------|
| 1 м³ | Ок |
| 82 м³ | Ок (граничное значение) |
| 83 м³ | Ошибка: превышение макс. объёма |
| 0 м³ | Ошибка: объём должен быть > 0 |
| Поле не заполнено при заполненном весе | Ок (перевозка только по весу) |
| 10.5 м³ (дробный) | Ок, если API поддерживает |

### 4.4. Дата погрузки

| Условие | Ожидаемый результат |
|---------|---------------------|
| Завтрашняя дата | Ок |
| Сегодняшняя дата | Ок |
| Вчерашняя дата | Ошибка: дата не может быть в прошлом |
| Дата через 6 месяцев | Ок (если нет ограничения) |
| Дата через 2 года | Зависит от бизнес-правил |
| 31 февраля | Ошибка: некорректная дата (датапикер должен блокировать) |

### 4.5. Кнопка «Рассчитать стоимость»

| Условие | Ожидаемый результат |
|---------|---------------------|
| Все поля валидны → нажать | 200 OK + цена в рублях |
| Не заполнены города → нажать | Ошибка валидации, кнопка disabled |
| Двойное нажатие | Один POST-запрос (idempotency) |
| API упал (500) | Тостер: «Ошибка расчёта, попробуйте позже» |
| API таймаут (30+ сек) | Тостер + кнопка разблокируется |
| Сеть недоступна | Ошибка на уровне браузера (fetch failed) |
| Маршрут не найден | Ошибка от API: «Маршрут не найден» |

### 4.6. Комбинаторные проверки

| Условие | Ожидаемый результат |
|---------|---------------------|
| Вес = 20 000 кг, объём = 82 м³ (оба max) | Ок, если оба в пределах |
| Вес = 1 кг, объём = 82 м³ (негабарит по объёму) | Ошибка или предупреждение |
| Вес не заполнен, объём заполнен | Зависит от бизнес-логики |
| Город ввода не совпадает с городом из БД (автокомплит) | Ошибка: выберите город из списка |
