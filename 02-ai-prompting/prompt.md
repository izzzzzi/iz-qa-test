# Prompt для LLM

## Контекст

Ты — QA-инженер, составляющий чек-лист для тестирования формы создания заявки на перевозку в сервисе «Умная Логистика Лайт» (MVP).

## Описание формы

Форма содержит следующие поля:

1. **Город отправления** — автокомплит/выпадающий список из базы данных
2. **Город назначения** — автокомплит/выпадающий список из базы данных
3. **Вес груза** — числовое поле, килограммы, максимум 20 000 кг
4. **Объём груза** — числовое поле, кубические метры, максимум 82 м³
5. **Дата погрузки** — датапикер, нельзя выбрать прошедшую дату
6. **Кнопка «Рассчитать стоимость»** — отправляет POST-запрос к API, возвращает цену в рублях или ошибку, если маршрут не найден

## Задача

Составь структурированный чек-лист для тестирования этой формы.

### Требования к чек-листу:

1. **Позитивные сценарии (happy path):** корректное заполнение всех полей, успешный расчёт
2. **Негативные сценарии:** ошибки валидации, некорректные данные
3. **Граничные значения:** для веса (max 20 000 кг), объёма (max 82 м³), даты
4. **API-проверки:** статус-коды, ошибки

### Формат вывода:

Категоризируй проверки по группам:
- Поля ввода (города, вес, объём, дата)
- Кнопка «Рассчитать стоимость»
- API и обработка ошибок
- UX и доступность

Для каждой проверки укажи:
- Шаг проверки
- Ожидаемый результат
- Тип (позитивный/негативный/граничный)

Учти, что это MVP — не добавляй избыточных проверок, которые не имеют смысла на этом этапе (например, нагрузочное тестирование, безопасность на уровне OWASP).
