# iz-qa-test

**QA Engineer (Middle+/Senior) — test assignment for Smart Logistics (Умная Логистика)**

## Решение

| Блок | Статус | Описание |
|------|--------|----------|
| [1. ЭТрН](01-etrn/README.md) | ✅ | Чек-лист бизнес-процесса (8 титулов, 60+ сценариев), проверки XSD, 3 баг-репорта |
| [2. AI Prompting](02-ai-prompting/README.md) | ✅ | Промпт → ответ LLM → ревью ответа (7 ошибок ИИ) + граничные условия |
| [3. SQL](03-sql/README.md) | ✅ | 3 запроса PostgreSQL: вторая зарплата, дубликаты email, клиенты без заказов |
| [5. 1С](05-1c/README.md) | ✅ | Запрос к регистру сведений + 8 найденных ошибок в коде |

## Структура репозитория

```
01-etrn/              ← Блок 1: ЭТрН — тест-кейсы, баг-репорты
  └── README.md
02-ai-prompting/      ← Блок 2: AI Prompting
  ├── README.md       ← ревью ответа ИИ + граничные условия
  ├── prompt.md        ← мой промпт к LLM
  └── llm-response.md  ← сырой ответ ИИ
03-sql/               ← Блок 3: SQL-запросы
  └── README.md
05-1c/                ← Блок 5: 1С — код-ревью + запрос
  └── README.md
```

## Примечания

- **Внешние ресурсы (Яндекс.Диск):** BPMN-диаграмма и API-спецификация требуют ручного прохода капчи. Чек-лист ЭТрН составлен на основе [официальной документации ФНС](https://www.nalog.gov.ru/rn77/about_fts/docs/14975031/) и [API Контур.Диадок](https://developer.kontur.ru/Docs/diadoc-api/docflows/waybill.html).
- **AI-ревью (блок 2):** Ключевая часть задания — не генерация чек-листа, а анализ того, где ИИ ошибся и что упустил.
