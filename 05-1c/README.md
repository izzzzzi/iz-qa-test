# Блок 5: 1С — Базовые вопросы про 1С

## Задача 1: Запрос к регистру сведений «Цены доставки»

**Условие:** Дан регистр сведений «Цены доставки» с полями:
- `Период` (дата)
- `ТипКлиента` (Юр/Физ)
- `ВесОт`, `ВесДо` (границы веса)
- `Стоимость`

Для заданных параметров (Дата заказа, Тип клиента, Общий вес) нужно вернуть актуальную стоимость.

### Уточнение бизнес-логики

Интервалы веса не должны пересекаться. Вес = 10 — должен брать интервал `до 10`, а не `от 10`. То есть интервалы вида:
- `[0, 10)` → вес 10 НЕ попадает
- `[10, 50)` → вес 10 попадает
- ...

Следовательно, границы `ВесОт` включаются, `ВесДо` — исключаются.

### Запрос на языке 1С

```1c
Функция ПолучитьСтоимостьДоставки(ДатаЗаказа, ТипКлиента, ОбщийВес)

    Запрос = Новый Запрос;
    Запрос.Текст =
    "ВЫБРАТЬ ПЕРВЫЕ 1
    |   ЦеныДоставки.Стоимость КАК Стоимость
    |ИЗ
    |   РегистрСведений.ЦеныДоставки КАК ЦеныДоставки
    |ГДЕ
    |   ЦеныДоставки.ТипКлиента = &ТипКлиента
    |   И ЦеныДоставки.Период <= &ДатаЗаказа
    |   И ЦеныДоставки.ВесОт <= &ОбщийВес
    |   И (ЦеныДоставки.ВесДо > &ОбщийВес ИЛИ ЦеныДоставки.ВесДо ЕСТЬ NULL)
    |УПОРЯДОЧИТЬ ПО
    |   ЦеныДоставки.Период УБЫВ";

    Запрос.УстановитьПараметр("ТипКлиента", ТипКлиента);
    Запрос.УстановитьПараметр("ДатаЗаказа", ДатаЗаказа);
    Запрос.УстановитьПараметр("ОбщийВес", ОбщийВес);

    Результат = Запрос.Выполнить();

    Если Результат.Пустой() Тогда
        Возврат 0; // Стоимость не определена
    Иначе
        Выборка = Результат.Выбрать();
        Выборка.Следующий();
        Возврат Выборка.Стоимость;
    КонецЕсли;

КонецФункции
```

**Пояснение ключевых моментов:**
1. `ВЫБРАТЬ ПЕРВЫЕ 1` — берём только первую (самую актуальную) запись
2. `ВесОт <= &ОбщийВес И (ВесДо > &ОбщийВес ИЛИ ВесДо ЕСТЬ NULL)` — корректная обработка интервала: нижняя граница включительно, верхняя — исключительно. Если верхняя граница не задана (NULL) — значит «до бесконечности»
3. `Период УБЫВ` — сортировка от новых к старым, чтобы первая запись была с максимальной датой ≤ дате заказа
4. Возвращаем `0`, если запись не найдена (вес вне известных интервалов или нет подходящей цены)

## Задача 2: Найти ошибки в коде

### Исходный код (из задания)

```1c
Функция РассчитатьСтоимость()
    СуммаВес = 0;
```

### Найденные ошибки (5+)

#### ❌ Ошибка 1: Параметр ТипКлиента передаётся некорректно (Severity: Critical)

```1c
Запрос.УстановитьПараметр("ТипКлиента", СправочникКлиенты.НайтиПоРеквизиту(...));
```

Проблема: `НайтиПоРеквизиту(...)` используется напрямую — это либо псевдокод, либо неполный вызов. В реальном коде метод `НайтиПоРеквизиту` не существует у справочника напрямую. Нужно использовать `Справочники.Клиенты.НайтиПоРеквизиту("Наименование", "Иванов")` или передавать ссылку на элемент справочника. Плюс нет обработки ситуации, когда клиент не найден.

**Исправление:**
```1c
ТипКлиентаСсылка = Справочники.ТипыКлиентов.НайтиПоКоду(КодТипаКлиента);
Если ТипКлиентаСсылка.Пустая() Тогда
    Возврат 0; // Тип клиента не найден — стоимость не определить
КонецЕсли;
Запрос.УстановитьПараметр("ТипКлиента", ТипКлиентаСсылка);
```

#### ❌ Ошибка 2: Параметр &СуммаВес используется как правая часть МЕЖДУ (Severity: Critical)

```1c
И &СуммаВес МЕЖДУ ЦеныДоставки.ВесОт И ЦеныДоставки.ВесДо
```

Проблема: `МЕЖДУ` в 1С — это `значение МЕЖДУ граница1 И граница2`. Проверяется `&СуммаВес >= ВесОт И &СуммаВес <= ВесДо`. Это работает, но **оператор МЕЖДУ включает обе границы** (`ВесОт <= СуммаВес <= ВесДо`). Как указано в условии — вес = 10 должен брать интервал `до 10`, а не `от 10`. То есть границы НЕ должны пересекаться, и верхняя граница должна быть исключающей. `МЕЖДУ` включает обе границы, что приводит к пересечению интервалов.

**Исправление (для условия из задачи — верхняя граница исключается):**
```1c
И ЦеныДоставки.ВесОт <= &СуммаВес
И (ЦеныДоставки.ВесДо > &СуммаВес ИЛИ ЦеныДоставки.ВесДо ЕСТЬ NULL)
```

#### ❌ Ошибка 3: Нет сортировки, но берётся первая запись (Severity: High)

В запросе есть `УПОРЯДОЧИТЬ ПО Период УБЫВ`, но нет `ВЫБРАТЬ ПЕРВЫЕ 1`. Запрос вернёт ВСЕ записи, удовлетворяющие условиям. Если на одну дату попадает несколько интервалов веса — результат будет содержать несколько строк. Код берёт первую строку из выборки, но без `ВЫБРАТЬ ПЕРВЫЕ 1` это может быть неопределённое поведение.

**Исправление:**
```1c
"ВЫБРАТЬ ПЕРВЫЕ 1
|   ЦеныДоставки.Стоимость КАК Стоимость
...
|УПОРЯДОЧИТЬ ПО
|   ЦеныДоставки.Период УБЫВ";
```

#### ❌ Ошибка 4: СуммаВес может прийти неверного типа (Severity: Medium)

`СуммаВес` — это числовая переменная, но в 1С типы должны строго соответствовать. Если `Строка.Вес` — это `Число`, то всё корректно. Но если `Вес` — это `Характеристика` или другой составной тип — может быть неявная ошибка преобразования.

Также нет проверки, что `СуммаВес > 0`. Если вес = 0, то доставка с весом от 0 до N не имеет смысла, но запрос может найти 0 как валидное значение.

**Исправление:**
```1c
СуммаВес = 0;
Для каждого Строка из ТабличнаяЧасть.Номенклатура Цикл
    ЗначениеВес = Строка.Вес;
    Если ТипЗнч(ЗначениеВес) = Тип("Число") Тогда
        СуммаВес = СуммаВес + ЗначениеВес;
    КонецЕсли;
КонецЦикла;

Если СуммаВес <= 0 Тогда
    Возврат 0; // Некорректный вес
КонецЕсли;
```

#### ❌ Ошибка 5: ДатаЗаказа — всегда ТекущаяДата(), а не реальная дата заказа (Severity: High)

```1c
Запрос.УстановитьПараметр("ДатаЗаказа", ТекущаяДата());
```

Проблема: Всегда передаётся текущая дата. Но заказ мог быть создан вчера, а расчёт стоимости происходит сегодня. Должна использоваться дата заказа, а не текущая дата. В контексте ЭТрН и логистики это критически важно — цена доставки определяется на дату заказа.

**Исправление:**
```1c
Запрос.УстановитьПараметр("ДатаЗаказа", ЭтаФорма.ДатаЗаказа); // или из реквизита документа
```

#### ❌ Ошибка 6: Возврат 0 вместо NULL при отсутствии стоимости (Severity: Medium)

```1c
Если Результат.Пустой() Тогда
    Возврат 0;
```

Проблема: Если стоимость не определена, функция возвращает 0. Но 0 — это валидное значение стоимости (бесплатная доставка). Нельзя отличить «стоимость не найдена» от «стоимость равна 0». В задании прямо сказано: «если вес выходит за max известный интервал — тогда стоимость не определена (вернуть NULL/0)».

**Исправление:**
```1c
Если Результат.Пустой() Тогда
    Возврат Неопределено; // чёткое отличие от "стоимость = 0"
```

#### ❌ Ошибка 7: Нет обработки несуществующего ТипКлиента в параметре (Severity: Medium)

Параметр `ТипКлиента` устанавливается через `СправочникКлиенты.НайтиПоРеквизиту(...)` — если клиент не найден, в параметр попадёт `Неопределено` или пустая ссылка. Запрос с пустой ссылкой в условии `ТипКлиента = &ТипКлиента` может отработать неожиданно — в 1С сравнение с пустой ссылкой может вернуть ложь, и результат будет пустым без явной ошибки. Диагностировать такую ситуацию сложно.

#### ❌ Ошибка 8: Нет обработки потенциального деления на периоды с одинаковой датой (Severity: Low)

Если в регистре есть две записи с одинаковой датой (например, 01.01.2025 — один интервал веса, и 01.01.2025 — другой), сортировка `Период УБЫВ` без дополнительного поля сортировки даёт нестабильный порядок. При добавлении `ВЫБРАТЬ ПЕРВЫЕ 1` будет выбрана произвольная запись. Нужна сортировка по дополнительному полю (например, по дате создания записи).

**Исправление:**
```1c
|УПОРЯДОЧИТЬ ПО
|   ЦеныДоставки.Период УБЫВ,
|   ЦеныДоставки.ДатаСоздания УБЫВ
```
